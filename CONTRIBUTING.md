## Guidelines

### Code Style
- Follow the existing formatting in .md files
- Use clear, descriptive variable names in MIPS code
- Add comments to explain complex logic
- Include execution tables for new code examples

### Documentation Style
- Use emojis consistently with the rest of the documentation
- Keep explanations simple and beginner-friendly
- Include step-by-step explanations for complex topics
- Add examples wherever possible

### Commit Messages
- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove, etc.)
- Keep the first line under 72 characters

## Code of Conduct

### Our Standards
- Be respectful and inclusive
- Welcome beginners and help them learn
- Provide constructive feedback
- Focus on what is best for the learning community

### Unacceptable Behavior
- Harassment or discriminatory language
- Trolling or insulting comments
- Publishing others' private information
- Any conduct that would be inappropriate in a professional setting
