    Study Plan¶

    üìÖ Timeline-Based Study Plans¶

    Choose the plan that fits your timeline!

    Plan A: 3-4 Days Before Exam¶

    Day 1: Fundamentals¶

    • ‚úÖ Read Labs 1-3
    • ‚úÖ Practice: Hello World, Basic I/O, Simple Arithmetic
    • ‚úÖ Write code from memory 3 times
    • Goal: Master basic syscalls and program structure

    Day 2: Loops & Arrays¶

    • ‚úÖ Read Labs 4-5
    • ‚úÖ Practice: Sum loops, Factorial, Array operations
    • ‚úÖ Understand branching (beq, bne, blt, etc.)
    • Goal: Comfortable with loops and array indexing

    Day 3: Advanced Topics¶

    • ‚úÖ Read Labs 6-7
    • ‚úÖ Practice: String operations, Procedures
    • ‚úÖ Understand jal/jr calling convention
    • Goal: Know how to work with strings and functions

    Day 4: Exam Prep¶

    • ‚úÖ Complete all PYQ Solutions
    • ‚úÖ Review Quick Reference
    • ‚úÖ Practice writing full programs
    • Goal: Exam-ready confidence

    Plan B: Last Night Before Exam (6-8 Hours)¶

    Hour 1-2: Core Patterns (30% of exam)¶

    • Read PYQ Solutions - Armstrong Number
    • Practice basic loop + condition pattern
    • Focus: Loop structure + if-else logic

    Hour 3-4: Procedures & Arrays (40% of exam)¶

    • Read Leaf Procedure
    • Read Array/String manipulation
    • Focus: jal/jr pattern + array indexing

    Hour 5-6: Recursion & Special Cases (20% of exam)¶

    • Read Polynomial Evaluation
    • Understand stack usage
    • Focus: Recursive pattern recognition

    Hour 7-8: Practice & Sleep! (10% of exam + 100% of energy)¶

    • Write 2-3 programs from memory
    • Review Quick Reference
    • Sleep 6-7 hours minimum (seriously!)

    Plan C: 2 Hours Before Exam (Emergency)¶

    Emergency Study Plan

    This is NOT recommended, but if you're here anyway...

    Priority 1 (45 min): Quick Reference - Memorize syscall numbers - Learn basic instruction patterns - Understand program structure

    Priority 2 (45 min): PYQ Question 1 & 2 - Armstrong number (covers loops + conditions) - Leaf procedure (covers procedures + arithmetic)

    Priority 3 (30 min): Skim Labs 1-5 Type 2 - Focus on execution tables - Look at final code only

    üéØ Learning Strategy¶

    For Each Topic:¶

    1. Read Code (5 min)
    2. Don't skip the explanation

    3. Follow the execution table

    4. Understand Logic (10 min)

    5. Why each instruction?

    6. What's the overall pattern?

    7. Write from Memory (15 min)

    8. Close the guide
    9. Write the full program

    10. Check against original

    11. Modify & Practice (10 min)

    12. Change the requirements
    13. What if input was different?
    14. Can you optimize it?

    What NOT to Do:¶

    ‚ùå Don't just read passively
    ‚ùå Don't memorize without understanding
    ‚ùå Don't skip the "simple" programs
    ‚ùå Don't stay up all night (you need sleep!)

    What TO Do:¶

    ‚úÖ Write code by hand
    ‚úÖ Test with mental execution
    ‚úÖ Focus on patterns, not individual programs
    ‚úÖ Get adequate sleep before exam

    üéì Success Checklist¶

    Before the exam, make sure you can:

    • Write a complete MIPS program structure (.data, .text, syscalls)
    • Read and print integers/strings
    • Implement loops with proper branching
    • Work with arrays (indexing, loading, storing)
    • Write and call procedures (jal/jr)
    • Handle strings (length, reverse, palindrome)
    • Understand register usage ($t, $s, $a, $v)
    • Exit programs properly
