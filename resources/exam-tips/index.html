    Exam Tips & Strategies¶

    üéØ Before the Exam¶

    The Night Before¶

    • Review Quick Reference one last time
    • Practice writing 2-3 programs from memory
    • Get 7-8 hours of sleep (seriously, this matters more than cramming!)
    • Prepare your materials (pen, calculator if allowed, student ID)

    Morning Of¶

    • Light breakfast (avoid heavy meals)
    • Review syscall numbers and basic patterns
    • Don't start new topics - just refresh what you know
    • Arrive 15 minutes early

    üìù During the Exam¶

    Time Management (for 60-min exam)¶

    Phase Time Activity
    Read 5 min Read ALL questions, choose order
    Plan 5 min Sketch logic for each question
    Code 40 min Write actual programs
    Review 10 min Check for common errors

    Question Selection Strategy¶

    1. Start with easiest question
    2. Builds confidence
    3. Guarantees some marks

    4. Warms up your brain

    5. Do high-value questions next

    6. Usually medium difficulty

    7. Best return on time investment

    8. Attempt hard questions last

    9. You have partial credit strategy ready
    10. Less time pressure if you can't complete

    üîç Step-by-Step Approach¶

    For Every Question:¶

    Step 1: Read Carefully (2 min)¶

    • What is the input?
    • What is the output?
    • What processing is needed?
    • Any special conditions?

    Step 2: Plan the Logic (3 min)¶

    Write pseudocode in comments: 

    # 1. Read number n
# 2. Initialize sum = 0
# 3. For i = 1 to n:
#       sum = sum + i
# 4. Print sum

    Step 3: Write Structure (1 min)¶

    .data
    # variables

.text
.globl main
main:
    # code

    li $v0, 10
    syscall

    Step 4: Implement Logic (10-15 min)¶

    • Follow your pseudocode
    • Add comments as you go
    • Use clear labels

    Step 5: Mental Testing (2 min)¶

    • Walk through with a simple example
    • Check register usage
    • Verify loop conditions

    ‚úÖ Common Errors Checklist¶

    Before submitting, check:

    Program Structure¶

    • .data section present?
    • .text section present?
    • .globl main declared?
    • Exit syscall at the end?

    Syscalls¶

    • Correct syscall numbers?
    • Right arguments in right registers?
    • String addresses loaded with la?

    Loops¶

    • Counter initialized?
    • Condition correct?
    • Counter incremented?
    • Jump to loop label?

    Arrays¶

    • Index multiplied by 4 (word size)?
    • Bounds checking?
    • Correct base address?

    Procedures¶

    • Arguments in $a registers?
    • Return value in $v registers?
    • Return with jr $ra?
    • Saved registers if needed?

    Strings¶

    • Null terminator check?
    • Byte operations (lb, sb)?
    • Buffer size declared?

    üí° Partial Credit Strategies¶

    If you're stuck, get maximum partial credit:

    Strategy 1: Write Structure + Comments¶

    Even if you can't complete the code: 

    .data
    num: .word 0
    result: .word 0

.text
.globl main
main:
    # Read input number
    li $v0, 5
    syscall

    # TODO: Process the number
    # Logic: 
    # - Initialize counter
    # - Loop while condition
    # - Calculate result

    # Print result
    li $v0, 1
    # la $a0, result  # Need to finish calculation
    # syscall

    # Exit
    li $v0, 10
    syscall

    You'll get credit for: - Correct structure - Understanding I/O - Showing you know the logic

    Strategy 2: Solve Simpler Version¶

    Can't do full question? Solve a simpler case:

    Question: Sum of even numbers from 1 to n
    If stuck: Write sum of ALL numbers from 1 to n

    Question: Recursive factorial
    If stuck: Write iterative factorial

    You still get 40-60% marks!

    Strategy 3: Comment What's Missing¶

    loop:
    # Check if number is prime
    # (Need to implement prime checking logic here)
    # For now, assuming all numbers pass

    addi $t0, $t0, 1
    blt $t0, $t1, loop

    Shows you understand flow even if you can't complete the logic.

    üéì Question Type Strategies¶

    Type 1: Basic I/O & Arithmetic¶

    Time: 5-10 min | Difficulty: ‚≠ê

    Strategy: - Fastest marks - do first! - Double-check syscall numbers - Test with simple values

    Example: Read two numbers, add them, print result

    Type 2: Loops & Conditions¶

    Time: 10-15 min | Difficulty: ‚≠ê‚≠ê

    Strategy: - Draw loop structure first - Check exit condition carefully - Verify counter increment

    Example: Sum of n numbers, Factorial

    Type 3: Arrays¶

    Time: 15-20 min | Difficulty: ‚≠ê‚≠ê‚≠ê

    Strategy: - Remember: index * 4 for word arrays - Use register for base address - Draw memory layout if confused

    Example: Find max in array, Reverse array

    Type 4: Procedures¶

    Time: 15-20 min | Difficulty: ‚≠ê‚≠ê‚≠ê

    Strategy: - Write caller and callee separately - Remember: jal to call, jr $ra to return - Use $a for args, $v for return

    Example: Function to compute x¬≤ + y¬≤

    Type 5: Strings¶

    Time: 20-25 min | Difficulty: ‚≠ê‚≠ê‚≠ê‚≠ê

    Strategy: - Always check for null terminator (\0) - Use lb/sb for byte operations - Draw string in memory if needed

    Example: String length, Palindrome check

    Type 6: Recursion¶

    Time: 25-30 min | Difficulty: ‚≠ê‚≠ê‚≠ê‚≠ê‚≠ê

    Strategy: - If possible, do iterative version for partial credit - Stack operations: push before call, pop after - Base case is critical

    Example: Recursive factorial, Fibonacci

    üö® Emergency Tactics¶

    If You're Running Out of Time:¶

    15 minutes left, 1 question incomplete? - Skip to exit syscall - Write TODO comments for missing parts - Move to next question

    10 minutes left, 2 questions incomplete? - Write structure + pseudocode for both - Complete I/O parts - Better to have 50% of both than 100% of one

    5 minutes left? - Check ALL questions have exit syscall - Verify syscall numbers - Add any missing comments - Do NOT start new code!

    üìä Marks Distribution (Typical)¶

    Component Marks % Focus
    Correct Logic 40% Main algorithm
    Correct Syntax 30% Valid MIPS code
    I/O Handling 15% Syscalls correct
    Comments 10% Code clarity
    Efficiency 5% Bonus for optimization

    Key Insight: Even with wrong logic, you can get 55% marks with good structure, syscalls, and comments!

    üéØ Final Checklist¶

    Right before submitting:

    • All questions have exit syscall
    • No syntax errors (labels, commas, register names)
    • Comments explain logic
    • Test with example values mentally
    • Name/ID on answer sheet
    • All pages numbered and attached
